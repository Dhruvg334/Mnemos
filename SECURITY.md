# Security policy

Please report suspected vulnerabilities privately to the repository owner rather than opening a public issue. Include the affected component, reproduction steps, impact, and any suggested mitigation.

Do not include live credentials, customer data, or exploit material in public discussions. Reports are triaged according to exploitability, tenant-boundary impact, approval bypass risk, provenance loss, and availability impact.

Supported security boundaries include authentication and role checks, tenant/site isolation, critical-action approval gates, scoped agent tools, secret-free public configuration, and synthetic public demonstration data.
